How do I create an add-in for Developers Pad?

Developers Pad includes full support for add-ins. If you are interested, please download the Developers Pad Add-In SDK available at DevelopersPad.com. If you have any futher queries, please email james@vbweb.co.uk

Why doesn't Developers Pad preview server-side scripts correctly?

In order for Developers Pad to preview server-side scripts, you need to make some configuration changes. At the moment, no intuitive way has been implemented to do this.. (Next version, promise!). For the moment, please follow these instructions:

1. Goto Tools|Options. Select IE Preview.
2. In the Local Path box, specify the local path of a web. You can seperate these with a ; (without the quotes).
3. In the Server box, specify the address of the server each local path corresponds to (it must be in the same order). For example

LocalPath: C:\inetpub\wwwroot\vbweb;c:\inetpub\wwwroot\devpad
ServerPath: http://vbweb.localnetwork;http://devpad.localnetwork

This maps C:\inetpub\wwwroot\vbweb to http://vbweb.localnetwork and so on. Please note no trailing slashes should be present.

Does Developers Pad support xyz programming language?

Developers Pad already includes support for a wide range of languages. If it doesn't currently have the language you want, you can add the support yourself! Simply go to Tools|Options, and click Languages. Next, click on the New Language button. You can now fill out the syntax information for your language. This will generate a new file in the DevPadPath\_syntax\ directory. When you have finished, please send the file to james@vbweb.co.uk so it can be included in the next version.

How can I decrease startup times?

Developers Pad now includes a number of 'optimization' settings. Currently, these need to be set manually in the registry. The entries for the following settings must be entered in the HKEY_CURRENT_USER\Software\VB and VBA Program Settings\DevPad\Settings registry key, and must be of REG_SZ type.

Key Name Valid Settings Description
SU_LoadDocument 0 (Off),1 (On) Should Developers Pad load a new document when loaded up?
SU_ShowProject 0 (Off),1 (On) Should Developers Pad load the project window when loaded up (irrespective of any other project settings)?
SU_InitAddins 0 (Off),1 (On) Should Developers Pad read the addins.ini file when starting up? NB: If you choose this option, you need to goto Tools|Manage Add-Ins in order for the Add-ins to be loaded and used.
