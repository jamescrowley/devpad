Source Code

The source code for Developers Pad is freely available at DevelopersPad.com (written in Visual Basic). However, this does not mean you are free to do whatever you like with it! Please see the source code section of the license (above) for your rights.

In order to keep control of the Developers Pad project, we have set down a few guidelines in order to ensure the source code remains consistent.

Naming Conventions

First of all, I need to admit that the naming conventions currently used in Developers Pad are not consistent. However, all source code written in the last few months is! You must follow these naming conventions for modified source.

1. Variables should have a one-letter suffix indicating their data type

Data Type Suffix
Boolean b
Double f
Integer n
Long l
Object o
String s
UDT or Data Struct (ie POINTAPI) t
Variant v

2. Indenting should be 4 spaces.

3. Private procedures should be suffixed with 'p'.

4. Private variables of a Class Module, Form etc (not individual procedures) should be suffixed with 'm_', and then the appropriate data type.

5. Comments should be ' with a space before the comment text. They should be on their own line where possible.

Example

Private m_sFileName As String
Private Sub pDisplayAll(bDo As Boolean)
Dim sNew As String
    ' do we need to do something?
    If bDo Then
        ' do something
    End If
End Sub
Public Sub ChangeSel()
    'public procedure
End Sub

Modifying the Source Code

All code changes that are submitted should not change or delete existing code. Existing codeshould be commented out, and the new code should have a header and footer comments so that it is obvious what was changed.

Example

'### [Name of Changer] [Date Changed] ###
'### [Description of Change and Reason Why] ###

'Old code commented out
'If FOO Then
'    blah
'Else
'    blahblah
'End If

'### New code ###

If FOOFOO Then
    blah
End If

'### End Change ###

Submitting the Source Code

Modified source code should be sent to devpad@vbweb.co.uk. Only new or changed files are to be sent. A short description should be included in the e-mail as well that details of what you are sending.

Your source code may, at our discretion, be integrated into the core program. Credit will be given if this is the case. We reserve the right to reject changes without reason.
